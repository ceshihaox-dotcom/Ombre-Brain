# Ombre-Brain · Rin 的优化版本说明

> 基于 [P0luz/Ombre-Brain](https://github.com/P0luz/Ombre-Brain) (`dcba1eb` @ 2026-04-19) fork 而来
> 维护者: [@ceshihaox-dotcom](https://github.com/ceshihaox-dotcom)
> 开源对齐版本: 2026-05-03

---

## 一句话介绍

这是 Ombre-Brain 的一个**分支版本**。**核心记忆机制——衰减公式、做梦、feel、记忆桶、情感权重——和原作者完全一致、未改动**。我主要做的是**前端体验和工具链**的全面优化,外加一组**检索/排序的命中精度**调优(尤其中文场景)。

如果用比喻:原作者建了一座漂亮的房子(记忆模型),我给它装修了内饰、加了几个家电(网页和工具)、把"找东西的索引"做得更顺手(检索)。**承重墙没动。** 检索层动了哪些、为什么,见下面第二部分的透明披露。

# 第一部分:网页界面 (重点)

## 1. 六个独立视图

原版只有一个 dashboard 网页,所有功能挤在一起。新版按用途拆成 6 个独立视图:

### 📅 时间线 `/v2/`
按时间倒着排列所有记忆,每天一张卡片,点开看那天的详情。右侧有迷你时间线可以快速跳到很久以前。

### 🔲 记忆格 `/v2/cells/`
日常用最多的视图。所有记忆铺成卡片或列表。
- 多种排序(重要度 / 时间 / "活跃度"score)
- 多种分组(按状态 / 按主题 / 不分组)
- 强大筛选(钉决 / Feel / 已内化 / 噪声 / 主题域 / 标签)
- **标签搜索框** — 1000+ 个标签也能直接搜
- 键盘导航(↑↓ 选 / 回车打开 / 空格预览 / x 选中)
- 多选 + 批量操作(改重要度 / 标噪声 / 删除)

### ✦ 记忆星图 `/v2/network/`
把所有记忆画成一个星空,相关的记忆之间有连线。看你的"记忆星座"。

### ⚙ 控制台 `/v2/console/`
管理类操作的入口,有四个子页:
- **导入工作台** — 把对话 / 笔记导入成记忆
- **配置** — 切换 LLM / 调衰减参数 / 改 Prompt
- **Breath 模拟** — 看记忆系统怎么"呼吸"
- **回收站** — 软删除的记忆,可恢复

### 📱 手机端 `/v2/mobile/`
独立的手机网页,可以"添加到主屏幕"当 App 用。
- 4 个 tab:记忆首页 / 审阅 / 日历 / 设置
- 暗夜模式
- 文件上传(粘贴或拖拽)
- 情感唤起罗盘(选一个心情找回相关记忆)
- 自定义日期选择器(原生 iOS 那个太丑,自己写了一个)

## 2. 工作流工具

### 🪄 导入工作台 — 可视化拆分

原版的导入像个黑盒:粘进去一段对话,等结果出来。新版让你看到全过程:

- **试跑模式** — 只解析前 N 段,先看看效果,不用怕一上来花太多 API 钱
- **实时进度** — 当前处理到第几段、累计花了多少美元
- **失败原因可见** — LLM 输出有问题时能看到原文,知道为什么挂了
- **支持 Claude Exporter 浏览器插件 JSON 格式** — 直接导入你和 Claude 的对话
- **多种来源** — 拖拽文件、粘贴文本、上传 ZIP 都行
- **跨天对话不再塌陷**(原版会把所有 turn 时间塌成一个)
- **chunk 内每行嵌入时间戳前缀**(`[2026-05-02 19:07] [用户] ...`)— 让 DIGEST_PROMPT 能给每条 item 准确推断 event_time。原版按 turn 切窗 ~10K token,但每行只标 `[用户]/[AI]` 不带时间。我加这条让发生时间能精确对应到具体段落,而不是整 chunk 共用一个时间戳

### 🔄 重新脱水 — 不满意可以重做

原版没这个功能。当 LLM 写的标题 / 摘要 / 标签 / 正文不满意时,新版可以:

- **一键让 LLM 重做**(默认只动元数据,正文不变)
- **勾选"同时重写正文"** — 用原始对话彻底重提炼正文
- **预览 → 编辑 → 接受 三步走**:
  1. 左右双栏对比旧版本和新版本
  2. 觉得不行可以编辑或重做
  3. 满意才真正写入
- **再也不会"做完不满意但原文已经没了"的尴尬**

技术细节:用了"主题锚点法",防止 LLM 把整段原文概括成流水账。

### 🤝 合并预览 — 防误合

系统检测到两条相似记忆建议合并时,新版会:
1. LLM 先写一份合并后的草稿
2. 给你看预览,可以编辑、重做、或拒绝
3. 接受才真正合并

### 📋 批量操作

桌面记忆格里:
- 多选记忆(点复选框或按 x 选中)
- 一次性改一批的重要度
- 一次性标噪声 / 取消噪声
- 一次性删除

### 🗑 软删除 + 回收站 — 防误删

原版删除就真的删了,找不回来。新版:
- 删除 = 移到回收站
- 设置 → 回收站 看到所有"删过"的
- 可以恢复,也可以"彻底删除"清空

### 🔌 多个 LLM 一键切换

原版每次换 LLM 都要改文件 + 重启服务。新版:
- 配置页加多组 API profile(一组 Claude / 一组 Gemini / 一组 DeepSeek...)
- 一键切换激活的 profile,**即时生效不用重启**
- "测试连接"按钮看 key 还有效吗、网络多快

### ✏️ Prompt 直接改 — 不用动代码

新版可以在网页里直接编辑 LLM 的系统提示词:
- 6 个 prompt 全部可视化编辑
- 单条恢复默认 / 全部恢复默认
- **"⇆ 对齐原作者版本" 按钮 — 一键切回上游 prompt** (这个特性专门为开源做的)

## 3. 智能筛选 — 找记忆变快

### 主题域 (Domain)
原版数据里一直存着 domain(比如"恋爱""内心""工作"),但 dashboard 只是个简单 chip 列表。新版**所有视图**(桌面 timeline / 记忆格 / 手机)都加了 domain 筛选,可多选 AND。

### 标签 (Tag)
原版没有标签筛选。新版:
- **桌面**:能搜 1000+ 个标签
- **手机**:点"🏷 标签"打开底部抽屉,搜索 + 多选
- 已选的永远显示在最上面(不会找不到怎么取消)

### 状态筛选
- 全部 / ★ 钉决 / ✦ 重要 / ❀ Feel / 已内化 / 待消化 / 我写的 / ⌀ 噪声

### 全文搜索
- 标题 / 摘要 / 正文 / 标签 都搜
- 桌面按 `/` 键快速聚焦

### 情感唤起罗盘 (Mood Evoke) — ⚗️ 实验性
- 在一个 2D 心情坐标里选(消极 ↔ 积极 × 平静 ↔ 激动)
- 找出最匹配那种心情的记忆
- LLM 用这种心情口吻把它们串成一段叙事
- ⚗️ **这是个还不太成熟的小玩具功能,没认真测过,图个好玩——别太当真**

## 4. 看清楚系统在干嘛 (可观测性)

### 💰 LLM 花了多少钱
原版没有花费提示。新版:
- 每次 LLM 调用显示 token 数 + 美元 + 人民币成本
- 右下角浮窗 widget 累计本次会话总开销
- 防止"导入一次几十块"的惊吓

### 📊 记忆的"分数"
- 列表里直接显示每条记忆的衰减分数 (decay score)
- 配置页能看公式怎么算的、调整 10 个权重参数
- **改完即时生效**,不用重启

### 🔍 进度反馈
- 导入 / 备份 / 重新脱水 等耗时操作都有进度横幅
- 失败有详细原因 + 建议怎么解决

### 🎨 视觉
- 月光紫主题 + 纸张感设计语言
- 暗夜模式
- 大量微交互动画(hover / 切换 / 加载)

# 第二部分:涉及到原作者代码的改动 (透明披露)

下面是涉及到底层代码的改动,我尽量解释为什么需要。如果你担心和原作者版本不一致,看完就知道每条都做了什么。

## A. Bug 修复 (建议回流上游)

### 1. 时区统一 UTC
原版后端混用本地时间和 UTC,在中国/日本等时区会差 8-9 小时。新版统一存 UTC,前端按你的时区显示。

### 2. 置顶桶可以标噪声
原版置顶 (`protected`) 的桶重要度被锁在 10,你想标"噪声"(改成 1)会被静默忽略。新版自动放开锁,让你能标。

### 3. 中文引号防 JSON 解析失败 (鲁棒性建议)
LLM 生成 JSON 时如果对话里有引号,容易因为引号嵌套导致 JSON 解析挂。新版在 prompt 里加了一条"用中文引号「」"做兜底,从此不挂。

## B. 字段重命名 (无法回退)

这部分是早期重构,新版前端到处用这些新字段,无法回退。但**完全兼容原版数据**(老桶自动迁移)。

- `pinned` 拆成 `protected`(防衰减) + `highlight`(浮现优先) — 原本一个字段管两件事容易混
- `digested` 改名 `internalized` — 跟"已归档"语义混
- 新增 `event_time`(事件实际发生时间,跟数据库写入时刻分开)
- 新增 `created_by`("ai" / "user",区分谁写的)

## C. 默认值 (已对齐上游)

| 字段 | 现在 | 之前我用过的 |
|---|---|---|
| `feel_score` | 50.0 (= 上游) | 0.0 |
| 自动归档 skip 名单 | `("permanent", "feel")` (= 上游) | 去掉 feel |
| 衰减 10 个参数 | 全是上游默认 | — |

我之前自己用时改过 feel 默认行为(因为我把 feel 当日记类型),开源前**全部对齐回上游**。如果你想跟我一样的体验,去配置页 → 权重配置改即可。

## D. Prompt (双轨制)

| Prompt | 上游有 | 代码默认 |
|---|---|---|
| `DEHYDRATE` | ✓ | 上游原版 |
| `DIGEST` | ✓ | 在上游基础上加了功能性规则(查看原文锚点 / 事件时间推断 / 重要度量表 / 防 LLM 保守打分),**去掉了我个人的 Rin/克 称谓**。注:重要度量表是按"重视亲密关系语境"调的,会让打闹/承诺/袒露类记忆分数偏高——见第二部分末尾说明 |
| `MERGE` | ✓ | **我精炼过的版本**(强调"沉淀成一条"而非拼接、反流水账、越合越短),比上游简版更稳。上游原版可一键对齐 |
| `REDEHYDRATE` | ✗ | 我独创 (上游没有这个功能) |
| `REGEN_CONTENT` | ✗ | 我独创 (主题锚点法,上游没有) |
| `ANALYZE` | ✓ | 上游原版 |

**配置页"⇆ 对齐原作者版本"按钮**一键把 DEHYDRATE/DIGEST/MERGE/ANALYZE 切回上游版。我独创的 REDEHYDRATE/REGEN_CONTENT 保留(上游没有,无可对齐)。

> 这套 prompt 是我按自己的习惯调的、在 DeepSeek 上实测过,**不一定适合所有人**——不满意配置页里直接改,或一键对齐原作者版。

## E. 新增的 API endpoint

下面这些都是"加法",不改原来的逻辑,只是新增能力:

- 重新脱水: `/api/bucket/{id}/redehydrate` + `/redehydrate-commit`
- 软删除: `/api/bucket/{id}/delete` + `/restore` + `/purge`
- 回收站: `/api/trash`
- API profile 管理: `/api/config/api/...`
- 衰减配置: `/api/decay-config` + `/reset`
- Prompt 编辑: `/api/prompts-config` + `/align-upstream` + `/reset`
- 自动备份: `/api/backup`
- Embedding 补缺: `/api/embeddings/backfill`

## F. 检索与排序精度 (中文场景优化)

原版检索在"中文长正文 + 关键词"场景下有些召回/排序短板,我做了一组优化。**再强调:这些只影响"搜索/浮现时哪些记忆被选中、怎么排序",不改衰减/做梦/feel/记忆桶/情感权重这些核心机制。**

**默认生效(开源版你拿到就是这样):**

- **关键词命中优先** — 任一字段命中你的搜索词,该记忆就会被选出,不会因为时间久、重要度低而被埋掉(上游是综合分过阈值才出)。解决"我明明写过却搜不到"。
- **正文权重调高** (`content_weight` 1.0→3.0) — 正文里写过的内容也能被搜到,不只靠标题/标签。
- **检索新近权重对齐上游** — 搜索排序里"偏向最近"的程度保持上游的温和值,老记忆不会在搜索里被快速埋掉。
- **中文分词更准** — 内置 jieba 分词,长句搜索不再失准。
- **feel(私密感受)只走专门入口** — 不参与普通关键词/浮现检索,只能通过 `domain='feel'` 读取(贴合"feel = 私密日记"的定位)。
- **钉决/永久参考桶不占自动浮现名额** — 它们本就常驻视野,日常检索只在标题精确命中时才再出现,避免重复刷屏。
- **噪声(你标过的)默认不出现在检索里**(可手动包含)。
- **移除了上游的"自动解决"** — 低重要、30 天没动的记忆不再被系统自动标记完成(避免替你做主),这类记忆存活期因此更长。

**可选的高级检索旋钮(默认全关 = 跟上游一致,配置页可开):** 严格关键词模式(jieba token 命中)、标题命中加分、关键词优先排序、温度偏置(让情感更暖的记忆更易浮现)。开了才改变召回,不开无影响。

**关于重要度评分(importance):** 我把它按**重视亲密关系**的语境调过——打闹、玩梗、承诺、袒露这类会给得比中性记忆高一些(它们是关系的证据,不是噪声)。所以你若觉得某些记忆分数偏高,**那是有意的,不是 bug**。不合你的用法?配置页可一键切回原作者的评分方式,或者来问我。

**关于批量导入:** 导入是 LLM 提炼,**可能漏掉个别片段**(尤其用较弱的模型时),同一份内容导入两次也会有差异——这是正常的,重要内容可手动补录。(我移除了上游一个专为 Claude"跳过亲密内容"设计的兜底机制:对 DeepSeek 这类"真漏不是故意漏"的模型,它反而效果不好。)

# 第三部分:每天自动备份 (防丢失)

新版自带一套备份机制:
- GitHub Actions 每天北京时间 03:00 自动运行
- 把所有 buckets/ 推到你的私人 GitHub 仓库
- 用纯 Python git (不需要系统装 git)
- 我之前因为环境变量错配丢过 7 天数据,这套是**长期解药**

部署时设 `OMBRE_BACKUP_REPO` / `OMBRE_BACKUP_TOKEN` 环境变量即可,详见 DEPLOY.md「自动备份配置」。
